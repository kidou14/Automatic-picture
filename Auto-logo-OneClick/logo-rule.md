# Logo 生成新规则方案（4词库随机组合版）

## 方案说明

### 与旧方案的区别

| 维度 | 旧方案（logo-oneclick.md） | 新方案（本文件） |
|------|--------------------------|----------------|
| 风格数 | 2套固定风格 | 20种风格自由组合 |
| 主题词 | 36个几何描述词 | 20个抽象概念词 |
| 形状 | 包含在主题词内 | 独立词库20个 |
| 颜色 | 嵌入风格描述中 | 主体色+背景色 各独立词库20个 |
| 理论组合数 | 72种 | 1,600,000种 |

### 生成逻辑

每次生成时，从五个词库各随机抽取 1 个词，拼装成最终 Prompt：

```
A（主题概念）× B（主体形状）× C1（主体颜色）× C2（背景颜色）× D（设计风格）= 单次 Prompt
```

## Prompt 模板

每次生成时，从 A/B/C1/C2/D 各随机抽1个词，代入以下模板：

```
请生成一个精致的512×512的App logo图标。

应用名称：{appName}

主题概念：{A}
主体形状：{B}
主体颜色：{C1}
背景颜色：{C2}
设计风格：{D}

要求：图形占画布80%面积，无文字，无字母，无水印，无底部衬底，高清渲染品质，主体与背景高对比度
```

### 示例组合（手动代入验证）

- A：能量爆发的瞬间
- B：放射状扇形展开
- C1：霓虹紫，内发光电波质感
- C2：纯黑，极致深邃无底色感
- D：霓虹发光，暗底内发光效果

生成 Prompt：
> 请生成一个精致的512×512的App logo图标。应用名称：MyApp。主题概念：能量爆发的瞬间。主体形状：放射状扇形展开。主体颜色：霓虹紫，内发光电波质感。背景颜色：纯黑，极致深邃无底色感。设计风格：霓虹发光，暗底内发光效果。要求：图形占画布80%面积，无文字，无字母，无水印，无底部衬底，高清渲染品质，主体与背景高对比度。

当确认词库满意后，在 `scripts/logo-server.js` 中做以下修改（参考当前第140-212行）：

```javascript
  "无限循环与永续",

const MAIN_SHAPES = [
  "双螺旋扭转体",
  "悬浮菱形嵌套结构",
  // ... 共20项

  "荧光青绿，高饱和电子发光感",
  "熔岩橙红，炽热燃烧渐变",
  // ... 共20项
];

// 词库 C2：背景颜色
const BG_COLORS = [
  "纯黑，极致深邃无底色感",
  "深空海军蓝，沉静宇宙感",
  // ... 共20项
];

// 词库 D：设计风格
const DESIGN_STYLES = [
  "macOS 磨砂玻璃半透明质感",
  "金属铸造感，高光冷硬",
  // ... 共20项

```

### 替换 buildPrompt 函数

```javascript
function buildPrompt(appName) {
  const pick = (arr) => arr[Math.floor(Math.random() * arr.length)];
  const A  = pick(THEME_CONCEPTS);
  const B  = pick(MAIN_SHAPES);
  const C1 = pick(BODY_COLORS);
  const C2 = pick(BG_COLORS);
  const D  = pick(DESIGN_STYLES);

  return `请生成一个精致的512×512的App logo图标。

应用名称：${appName}

主题概念：${A}
主体形状：${B}
主体颜色：${C1}
背景颜色：${C2}
设计风格：${D}

要求：图形占画布80%面积，无文字，无字母，无水印，无底部衬底，高清渲染品质，主体与背景高对比度`;
}
```

> 如需保留切换旧方案的能力，可在接口增加 `rule=new` 参数，判断后走不同的 buildPrompt 逻辑。
